HTML Tables: Data, Not Layout

Main rule

Tables are for tabular data only. Never use tables for page layout. Use CSS Grid or Flexbox instead.

List of examples

1. Basic Table Structure

Monthly Expenses
Month Food Transport Total
January $300 $100 $400
February $280 $120 $400 
            <table>
    <caption>Monthly Expenses</caption>
    <tr>
        <th scope="col">Month</th>
        <th scope="col">Food</th>
        <td>$300</td>
    </tr>
</table>

2. Semantic Table (thead, tbody, tfoot)

Project Team Schedule
Task Assignee Deadline
Design mockups Alex Mar 15
Frontend development Maria Mar 25
Total tasks 2 
            <table>
    <thead>...</thead>  <!-- Column headers -->
    <tbody>...</tbody>  <!-- Main data -->
    <tfoot>...</tfoot>  <!-- Summary -->
</table>

3. Cell Spanning (colspan, rowspan)

Company Departments
IT Department
Developers Frontend: 5
Backend: 3
Designers UI/UX: 2 
            
    <th colspan="2">IT Department</th>  <!-- Merge 2 columns -->
    <th rowspan="2">Developers</th>  <!-- Merge 2 rows -->

Note: Use sparingly - complex spanning breaks accessibility.

4. What NOT to do (Common Mistakes)

Using table for layout:

                <table>
    <tr>
    <td>Sidebar</td>
    <td>Main Content</td>  <!-- Use CSS Grid! -->
    </tr>
</table>

Missing accessibility:

<th>Name</th> <!-- Missing scope="col" -->

Empty cells without meaning:

<td></td> <!-- Use: <td>-</td> or aria-label -->

5. Responsive Table Pattern

On mobile, table scrolls horizontally:

Name Email Phone Department Role Start Date
John Smith john@company.com (123) 456-7890 Engineering Frontend Developer 2023-01-15 
            <div class="table-container">
    <table>...</table>
</div>

<style>
    .table-container {
        overflow-x: auto;
        -webkit-overflow-scrolling: touch;
    }
</style>

Working with Images in HTML

Main rule

Every image needs alt text and dimensions. Accessibility first, performance second.

List of examples

1. Basic Image (All Required Attributes)

<img src="book.jpg" alt="Open book on table" width="400" height="300" loading="lazy">

Why: Alt for accessibility, dimensions prevent layout shift, lazy loading for performance.

2. Image with Caption (<figure> & <figcaption>)

            <figure>
    <img src="mountain.jpg" alt="Mountain landscape" width="400" height="300">
    <figcaption>Figure 1: Sunset in the Swiss Alps</figcaption>
</figure>

3. Responsive Images (srcset & sizes)

<img src="photo-400.jpg"
     alt="Description"
     width="400" height="300"
     srcset="photo-200.jpg 200w,
             photo-400.jpg 400w,
             photo-800.jpg 800w"
     sizes="(max-width: 600px) 200px,
            (max-width: 1200px) 400px,
            800px">

4. Decorative Image (CSS Background)

This content has a decorative background image.

No alt needed here - it's just decoration.

.hero {
    background-image: url('pattern.svg');
    background-size: cover;
}


<div class="hero" role="img" aria-label="Decorative pattern">
    <!-- Content -->
</div>

5. Common Mistakes

Missing alt text:

<img src="cat.jpg"> ← BAD: No alt!

Generic alt:

<img src="chart.jpg" alt="image"> ← BAD: Too vague!

No dimensions:

<img src="product.jpg" alt="Product"> ← BAD: Causes layout shift!

Using <img> for decoration:

<img src="pattern.png" alt=""> ← BETTER: Use CSS background!

6. Image Format Guide

  • JPEG: Photos (lossy, good compression)
  • PNG: Graphics with transparency (lossless)
  • WebP: Modern format (better than JPEG/PNG)
  • SVG: Vector graphics (scales infinitely)
  • AVIF: Next-gen (best compression, limited support)

7. Quick Checklist for Every Image

  • Added meaningful alt text?
  • Specified width and height?
  • Optimized file size?
  • Chose right format (WebP/JPEG/PNG/SVG)?
  • Added loading="lazy" for below-fold images?
  • Have permission to use this image?
